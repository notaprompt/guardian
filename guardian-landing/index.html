The Problem

Chat sessions evaporate.
Your thinking shouldn't.

■

You solved this problem last month. You know you did. But the conversation is gone, the context is gone, so you rebuild from scratch.

■

Note apps add organizational overhead. Chat wrappers forget everything. Second-brain tools require you to be the librarian of your own thoughts.

■

Every insight you re-derive is time spent on something you already figured out. The solution is persistent memory that retrieves what you need, when you need it.

Guardian stores, compresses, and retrieves everything you think through.
On your machine. Searchable. Yours.

Capabilities

What it does.

01

Persistent Memory

Every conversation stored locally. Three-level compression: session summaries, extracted patterns, distilled principles. Raw detail when you need it. Compressed knowledge when you don't.

Full-text search Semantic retrieval Session continuity
02

Session Instruments

Token usage. Session depth. Integration queue status. The information you need about your current session, visible without demanding attention.

Session telemetry Integration queue Usage tracking
03

Chat with Context

Multi-provider chat with automatic context injection from your memory. Past conversations surface when relevant. You never have to say "remember when we talked about..."

Context injection Multi-provider Thinking display
04

Integrated Terminal

Real PTY, not emulation. Docks inline or floats as a separate window. Command history captured and searchable. No context-switching between terminal and chat.

Real PTY Dock / float Searchable history
05

Notes

Three modes: scratch for quick capture, structured for documents, journal for longer-form writing. Full version history. Mark anything sovereign to exclude it from chat context.

Scratch capture Markdown Version history
06

Knowledge Graph

Entities and relationships extracted from conversations automatically. Stored in SQLite, visualized with force-directed layout. See the structure of what you've been working on.

Entity extraction Relationship mapping Interactive visualization

How It Works

Local-first.
Sovereign by default.

Guardian runs entirely on your machine. SQLite for storage. Full-text search for retrieval. No cloud dependency for core functionality. Cloud providers are optional -- for chat, not for memory.

Your memory shapes what gets stored. What gets stored shapes what Guardian retrieves. What Guardian retrieves shapes your next conversation. The system gets more useful the more you use it.

Every feature answers one question: does this help you find and build on what you've already figured out? If not, it doesn't ship.

Everything in One Place

Less to Hold in Your Head

Context injection means past work shows up when relevant. You think forward. Guardian handles the retrieval.

Nothing Gets Lost

Every conversation, every note, every artifact is stored and versioned. Deleting is your choice, never the system's.

Your Data Stays Here

No telemetry. No phone home. Local SQLite. Cloud providers are for chat -- your memory never leaves your machine.
